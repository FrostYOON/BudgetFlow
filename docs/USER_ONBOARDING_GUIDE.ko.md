# BudgetFlow 사용자 온보딩 가이드

## 이 문서는 누구를 위한 문서인가요?
- 이 문서는 BudgetFlow를 처음 사용하는 사람을 위한 안내서입니다.
- 모바일과 데스크탑 모두 기준으로 설명합니다.
- 가장 추천하는 시작 방식은 `개인 예산부터 시작`하고, 필요할 때 `공유 워크스페이스`로 확장하는 흐름입니다.

## 1. 첫 로그인 후 시작 흐름
1. `/sign-up`에서 계정을 생성합니다.
2. 회원가입이 끝나면 바로 앱 안으로 진입합니다.
3. 혼자서 먼저 사용해보고 싶다면 기본 개인 워크스페이스로 시작하면 됩니다.
4. 커플, 가족, 룸메이트처럼 함께 쓰려면 앱 안에서 공유 워크스페이스를 만들면 됩니다.

## 2. 메인 메뉴 구조
- `Dashboard`: 이번 달 상태 요약, 예산 건강도, 최근 변화, 정산 바로가기
- `Spend`: 수입/지출 입력, 필터, CSV 가져오기, 수정, 삭제, 복구
- `Budgets`: 월 총액 설정, 카테고리별 예산 배분, 템플릿/복사 도구
- `More`: 반복 거래, 리포트, 알림, 설정, 보조 기능

### 모바일 메뉴 사용 팁
- 하단 도크를 눌러 4개의 주요 영역을 이동합니다.
- 메인 앱 화면에서는 좌우 스와이프로 `Dashboard`, `Spend`, `Budgets`, `More`를 빠르게 넘길 수 있습니다.
- 버튼, 입력창, 링크, 가로 스크롤 영역에서는 스와이프 이동이 작동하지 않도록 되어 있어 폼 사용이 안정적입니다.

## 3. 처음 10분 추천 사용 순서
1. `Spend`에서 지출 거래 1개를 추가합니다.
2. 수입 거래도 1개 추가합니다.
3. `Budgets`에서 이번 달 총 예산을 입력합니다.
4. 자주 쓰는 카테고리 몇 개만 먼저 배분합니다.
5. `Dashboard`로 돌아와 월 요약이 반영됐는지 확인합니다.
6. 다른 사람과 함께 쓸 예정이면 공유 워크스페이스를 만듭니다.
7. `Settings`에서 테마, 로케일, 타임존, 통화 관련 설정을 정리합니다.

## 4. 화면별 사용 방법

## Dashboard
- 월 단위로 전체 상황을 확인하는 메인 화면입니다.
- 상단 액션 영역에서 거래 추가, 정산 보기, 리포트 열기, 월 이동을 빠르게 할 수 있습니다.
- 요약 카드는 예산 사용 상태, 공유 지출, 총 예산, 인사이트 개수를 보여줍니다.
- 거래 캘린더로 날짜별 활동을 확인할 수 있습니다.
- 인사이트 영역은 경고나 이상 징후를 가장 빨리 확인하는 곳입니다.

## Spend
- 가장 자주 쓰게 되는 실사용 화면입니다.
- `Add entry`로 수동 거래를 입력합니다.
- `Filters`로 거래 유형, 공개 범위, 카테고리, 결제자, 금액 범위를 좁힙니다.
- `Import CSV`는 기존 데이터를 한 번에 옮길 때 사용합니다.
- `Expense / Income / All`, `Shared / Personal / All` 토글로 리스트를 빠르게 바꿉니다.
- 거래를 누르면 상세를 보거나 수정할 수 있습니다.
- 잘못 삭제했으면 복구 배너에서 바로 되돌릴 수 있습니다.

### Spend 사용 팁
- 금액, 날짜, 카테고리, 결제자를 먼저 정확히 넣는 게 중요합니다.
- 메모는 짧고 실용적으로 적는 편이 좋습니다.
- 정산에 반영해야 하는 거래만 shared로 두는 게 좋습니다.

## Budgets
- 먼저 월 총액부터 정합니다.
- 그 다음 실제로 자주 쓰는 카테고리부터 배분합니다.
- 모바일에서는 한 번에 너무 많은 정보가 보이지 않도록 낮은 우선순위 기능이 접혀 있습니다.
- `Plan categories`는 예산 배분 작업 영역입니다.
- `Open categories`는 아직 예산이 없는 카테고리 목록입니다.
- 템플릿과 복사 도구는 한 달 정도 구조가 잡힌 뒤 쓰는 것이 좋습니다.

### Budgets 사용 팁
- 카테고리 배분 전에 월 총액을 먼저 정하세요.
- 식비, 월세, 교통, 구독, 공과금처럼 고정성이 높은 항목부터 배분하면 편합니다.
- 지난달 복사는 구조가 안정된 뒤에 쓰는 것이 좋습니다.

## Settlements
- 공유 지출을 정리할 때 쓰는 화면입니다.
- 멤버별 잔액과 추천 송금 방향을 확인할 수 있습니다.
- 개인 전용 워크스페이스에서는 중요도가 낮습니다.

## Recurring
- 구독료, 월세, 급여, 반복 이체 같은 정기 거래를 자동화할 때 사용합니다.
- 처음부터 너무 많이 만들지 말고, 확실한 규칙 1~2개부터 시작하는 것이 좋습니다.
- 규칙을 만들거나 바꾼 뒤에는 최근 실행 결과와 실패 상태를 확인하는 것이 안전합니다.
- 수동 실행은 규칙이 제대로 동작하는지 즉시 확인할 때 유용합니다.

## Reports
- 매일 쓰는 화면이라기보다 월말 정리용 화면입니다.
- 스프레드시트나 외부 검토가 필요하면 export를 사용합니다.
- print view는 정리된 형태로 검토할 때 좋습니다.

## Notifications
- 알림은 보조 정보 확인 화면입니다.
- 핵심 작업 공간이라기보다 후속 확인용으로 생각하면 됩니다.

## Settings
- 앱 자체를 관리하는 화면입니다.
- `Appearance`에서 `Auto`, `Light`, `Dark`를 고를 수 있습니다.
- 계정 이름, 로케일, 타임존, 비밀번호 등을 여기서 관리합니다.
- 워크스페이스 프로필, 초대, 세션, 소유자 전용 기능도 여기서 다룹니다.

## 5. 공유 워크스페이스 사용 흐름
1. 온보딩에서 공유 워크스페이스를 생성합니다.
2. `Settings`를 엽니다.
3. 공유 설정 영역에서 초대 이메일을 보냅니다.
4. 생성된 참여 링크를 상대에게 공유합니다.
5. 초대받은 사용자가 로그인하거나 새 계정을 만듭니다.
6. 초대를 수락하면 워크스페이스 스위처에 해당 공간이 나타납니다.
7. 이후 shared 거래는 정산과 공유 요약에 반영됩니다.

## 6. 테마와 모바일 사용 방식
- 테마 설정은 `Settings > Appearance`에 있습니다.
- 모바일은 한 손 사용 기준으로 설계되어 있습니다.
- 하단 도크 하나
- 화면당 대표 액션 하나
- 압축된 요약 카드
- 주요 탭 간 좌우 스와이프 이동

## 7. 대표적인 사용 패턴

## 개인 사용자
1. 평소에는 `Spend`를 중심으로 사용합니다.
2. 한 달에 한 번 `Budgets`를 정리합니다.
3. 주간 단위로 `Dashboard`를 확인합니다.
4. 월말에는 `Reports`로 정리합니다.

## 커플 / 가족 사용자
1. `Spend`에서 shared 거래를 입력합니다.
2. `Dashboard`에서 예산과 공유 상태를 같이 봅니다.
3. 송금이 필요할 때 `Settlements`를 엽니다.
4. 구독료나 월세는 `Recurring`으로 자동화합니다.

## 8. 초반에 피하면 좋은 것
- 첫날부터 반복 거래를 너무 많이 만들지 마세요.
- 실제 거래 입력 전에 카테고리를 과하게 세분화하지 마세요.
- 안정적인 한 달이 생기기 전까지 템플릿에 과하게 의존하지 마세요.
- 실제로 개인 지출인 항목까지 모두 shared로 두지 마세요.

## 9. 가장 단순한 이해 방식
- `Dashboard`는 무슨 일이 바뀌었는지 보는 곳입니다.
- `Spend`는 가장 자주 일하는 곳입니다.
- `Budgets`는 계획을 세우는 곳입니다.
- `More`는 보조 도구가 모인 곳입니다.
