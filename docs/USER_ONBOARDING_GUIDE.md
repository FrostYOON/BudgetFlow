# BudgetFlow User Onboarding Guide

## Who This Guide Is For
- This guide is for someone using BudgetFlow for the first time on mobile or desktop.
- The recommended first path is `personal budget first`, then `shared workspace` only when needed.

## 1. First Sign-In Flow
1. Open the app and create an account from `/sign-up`.
2. After sign-up, BudgetFlow sends you straight into the app.
3. If you only want to track your own money at first, you can start with the default personal setup.
4. If you need a couple, family, or roommate setup, open onboarding from the app and create a shared workspace.

## 2. Main Navigation
- `Dashboard`: month summary, budget health, recent signals, and settlement shortcut
- `Spend`: daily transaction entry, filters, CSV import, edit, delete, restore
- `Budgets`: monthly total, category allocations, template and copy tools
- `More`: recurring, reports, notifications, settings, and secondary tools

### Mobile Navigation Tips
- Use the bottom dock to move between the 4 primary sections.
- Swipe left or right on the main app surface to move between `Dashboard`, `Spend`, `Budgets`, and `More`.
- Swipe navigation is ignored on buttons, inputs, links, and horizontal scrollers so forms should still feel stable.

## 3. Recommended First 10 Minutes
1. Open `Spend` and add one expense transaction.
2. Add one income transaction.
3. Open `Budgets` and set the total monthly budget.
4. Allocate a few categories you use often.
5. Return to `Dashboard` and confirm the month summary reflects the new data.
6. If you plan with another person, create or switch to a shared workspace.
7. Open `Settings` and choose your theme mode, locale, timezone, and currency context.

## 4. How To Use Each Area

## Dashboard
- Use this as the monthly control surface.
- The top area is for quick actions: add a transaction, open settlements, open the report, move month.
- The summary cards tell you budget use, shared spend, total budget, and insight count.
- The transaction calendar helps you inspect activity by date.
- The insights panel is the fastest way to see warnings or unusual budget behavior.

## Spend
- This is the fastest screen for day-to-day usage.
- Use `Add entry` for manual transaction input.
- Use `Filters` to narrow by type, visibility, category, payer, or amount range.
- Use `Import CSV` for bulk entry when moving existing data in.
- Tap `Expense / Income / All` and `Shared / Personal / All` to quickly change the list view.
- Open an existing item to view or edit it.
- If you delete the wrong item, use the restore banner immediately.

### Best Practice For Spend
- Enter the amount, date, category, and payer first.
- Keep memo short and practical.
- Use shared visibility only when the expense should affect settlement.

## Budgets
- Start with the monthly total first.
- Then allocate categories you actually use often.
- On mobile, BudgetFlow hides lower-priority controls until you open them.
- `Plan categories` is for allocation work.
- `Open categories` contains categories with no allocation yet.
- Template and copy tools are useful after you already have one stable month.

### Best Practice For Budgets
- Set the monthly total before you distribute category amounts.
- Prioritize groceries, rent, transport, subscriptions, and utilities first.
- Use previous-month copy only after you trust last month’s structure.

## Settlements
- Use this only for shared money.
- It shows shared expense totals, balance by member, and suggested transfers.
- If your workspace is personal-only, this section matters less.

## Recurring
- Use recurring rules for subscriptions, rent, salary, and repeat transfers.
- Start with one or two stable rules instead of automating everything at once.
- Check recent runs and failures after you create or change a rule.
- Manual run is useful when you want to verify rule behavior immediately.

## Reports
- Reports are for monthly review, not daily input.
- Use export when you need to move data into spreadsheets or external review.
- Use print view for a cleaner summary layout.

## Notifications
- Notifications are a secondary surface for reminders and updates.
- Treat them as follow-up context, not as your primary work area.

## Settings
- This is where you manage the app itself.
- Use `Appearance` to choose `Auto`, `Light`, or `Dark`.
- Update account name, locale, timezone, and password here.
- Manage workspace profile, invites, sessions, and owner-only controls here.

## 5. Shared Workspace Flow
1. Open onboarding and create a shared workspace.
2. Open `Settings`.
3. Send an invite email from the shared space section.
4. Share the generated join link.
5. The invited user signs in or creates an account.
6. After they accept, the workspace appears in the switcher.
7. Shared transactions start affecting settlement and shared summaries.

## 6. Theme, Mobile, And App Feel
- Theme mode now lives in `Settings > Appearance`.
- Mobile is designed around one-hand usage:
- one bottom dock
- one main action at a time
- compressed summary cards
- swipe navigation between main tabs

## 7. Common Usage Patterns

## Personal User
1. Use `Spend` daily.
2. Use `Budgets` once per month.
3. Use `Dashboard` for weekly review.
4. Use `Reports` at month end.

## Couple Or Family User
1. Add shared transactions from `Spend`.
2. Review `Dashboard` for budget and shared status.
3. Open `Settlements` when transfers are needed.
4. Use `Recurring` for subscriptions and rent.

## 8. What To Avoid Early On
- Do not create too many recurring rules on day one.
- Do not over-model categories before entering real transactions.
- Do not rely on templates before one clean month exists.
- Do not turn every expense into shared if it is actually personal.

## 9. Simple Mental Model
- `Dashboard` tells you what changed.
- `Spend` is where you work most often.
- `Budgets` is where you plan.
- `More` is where supporting tools live.
